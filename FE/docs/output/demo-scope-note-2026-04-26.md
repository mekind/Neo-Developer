# Demo Scope Note — 2026-04-26

## Summary
- FE 문서 컨텍스트를 데모 / 프로토타입 기준으로 명시했다.
- 이후 문서와 구현 판단은 production hardening보다 데모 전달력과 범위 명확성을 우선한다.

## What changed
- `FE/docs/config.md`에 demo-oriented 목적과 product stage 제약을 추가했다.
- `FE/docs/wiki/topics/frontend-workspace-overview.md`에 현재 FE 트랙이 demo / prototype 용도임을 명시했다.
- `FE/docs/log.md`에 해당 결정의 변경 이력을 남겼다.

## Why this matters
- 이후 에이전트가 문서/구현 수준을 과도하게 production 기준으로 끌어올리지 않도록 하기 위함이다.
- 해커톤/데모 맥락에서 필요한 의사결정 기준을 FE 문서 루트에 고정하기 위함이다.
