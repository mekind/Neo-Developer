# Stage 1: Problem Framing & Analysis

## 서비스명: MyClaw

---

## 1. 문제 정의 (5W1H)

| 항목 | 정의 |
|------|------|
| **What** | 비개발자가 대화만으로 AI 에이전트를 정의하고, Gather town 스타일 공간에서 도트 캐릭터로 만나 대화하는 플랫폼 |
| **Who** | 비개발자 (코딩 없이 에이전트 만들고 싶은 사람) |
| **Why** | 기존 에이전트/챗봇 도구는 설정이 복잡해서 비개발자가 포기함 |
| **Where** | 웹 기반 Gather town 스타일 공간 |
| **When** | 반복 업무 자동화, 정보 수집, 심심할 때 대화 등 |
| **How** | Claude 세션에서 대화로 정의 → OpenClaw가 런타임 처리 |

---

## 2. 핵심 Pain Point

| 페르소나 | Pain Point | 현재 해결책 |
|---------|-----------|------------|
| 수진 (마케터) | 매일 경쟁사 블로그 수동 확인 (15분/일) | 북마크 돌기 |
| 동현 (쇼핑몰 대표) | 반복 문의 응대 (20분/일), 챗봇 설정 포기 | 복붙 |
| 은서 (회사원) | 뭘 만들지 모름, 예시 없으면 이탈 | 그냥 나감 |

## 3. 서비스 구조

```
┌─────────────────────────────────────────────────────────────┐
│  MyClaw — "OpenClaw 몰라도 재밌게 쓰는 개인맞춤 에이전트"    │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  [Claude 세션]                                              │
│  ├─ 온보딩 Agent: 사용자 정보 clarify → Memory 저장         │
│  └─ Persona Builder: 자연어 → 에이전트 정의 생성            │
│                                                             │
│  [OpenClaw 세션]                                            │
│  ├─ 에이전트 런타임: SOUL.md 기반 동작                      │
│  └─ Tool(Skill) 실행: insane-search 등                      │
│                                                             │
│  [Gather town 스타일 공간]                                  │
│  ├─ 사용자 캐릭터 이동                                      │
│  ├─ 에이전트 캐릭터 배치                                    │
│  └─ 다가가면 대화 시작                                      │
│                                                             │
│  [Memory 시스템 - llm-wiki 패턴]                            │
│  ├─ 사용자 정보 (profile, preferences)                      │
│  ├─ 에이전트 정의 (persona, SOUL, config)                   │
│  └─ 컨텍스트 축적 (interests, history)                      │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

## 4. 범위 정의

### In Scope
- 온보딩 Agent (사용자 정보 수집)
- Persona Builder Agent (에이전트 정의 생성)
- 에이전트 런타임 (OpenClaw 기반)
- Gather town 스타일 UI
- Memory 시스템 (llm-wiki 패턴)

### Out of Scope
- 시스템 아키텍처 상세 (사용자가 별도 정의)
- 요금제 상세
- 도트 캐릭터 이미지 생성 방식 (미정)

---

## 5. 핵심 가정

| 가정 | 근거 |
|------|------|
| OpenClaw가 multi-agent routing 지원 | 공식 문서 확인 |
| Claude 세션이 별도로 동작 가능 | 사용자 확인 |
| 비개발자가 대화만으로 에이전트 정의 가능 | Persona Builder가 clarify |
| Tool(Skill) 목록은 사전 정의됨 | 사용자 확인 (insane-search 등) |

---

## 6. 핵심 차별점

| 기존 서비스 문제 | MyClaw 해결책 |
|-----------------|--------------|
| 챗봇 설정 복잡 (시나리오 빌더) | 대화로 정의 |
| 기술 용어 필요 | 자연어로 말하면 됨 |
| 재미 없음 | Gather town 스타일 공간 + 도트 캐릭터 |
| 매번 같은 질문 | Memory가 쌓여서 질문 감소 |
