## QA-O-01: 기본 온보딩 완료

### 목적

이름 → 목적 → 관심사 → 완료 안내까지 정상 경로를 확인한다.

### 전제 조건

- Memory: 비어 있음 (신규 사용자)

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 짧은 첫 인사 + 이름 질문 **하나만** |
| 1 | `민지라고 불러줘요` | 이름 수용 → 목적 질문 **하나만** |
| 2 | `중요한 정보만 편하게 받고 싶어요` | 목적 수용 → 관심사 질문 + skip 안내 |
| 3 | `IT랑 스타트업이요` | 관심사 수용 → 완료 메시지 + 다음 단계 안내 |

### 기대 결과

- 총 질문 수: 3개 (이름, 목적, 관심사)
- 완료 메시지에 닉네임(`민지`) 포함
- 다음 단계(에이전트 만들기 공간) 안내 포함
- tech_level 내부 추론됨 (사용자에게 노출 안 됨)

### Pass 기준

- [ ] 한 턴에 질문이 1개만 포함됨
- [ ] 첫 인사가 3문장 이하로 짧음
- [ ] 기술 수준을 직접 묻지 않음
- [ ] 완료 메시지에 닉네임 포함
- [ ] 다음 단계 전환 안내가 있음
- [ ] profile.md 출력 스키마의 모든 필수 필드 충족
- [ ] `content.identity.name` = `"민지"`
- [ ] `content.context.purpose`에 사용자 원문 보존
- [ ] `content.context.tech_level`이 `low`/`medium`/`high` 중 하나

### Fail 기준

- [ ] 한 턴에 2개 이상 질문이 있음
- [ ] 기술 수준을 직접 묻는 표현 존재 ("개발 잘하세요?", "초급/중급?" 등)
- [ ] 관심사에 skip 경로 안내 없음
- [ ] 완료 후 Persona Builder 행동 발생
- [ ] tech_level이 사용자에게 노출됨

### 캡처할 증거

- 전체 대화 로그 (턴별)
- 최종 산출 profile.md 구조화 데이터
- tech_level 추론값

## QA-O-02: 관심분야 스킵

### 목적

사용자가 관심사를 skip해도 온보딩이 정상 완료되는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `준` | 이름 수용 → 목적 질문 |
| 2 | `그냥 재밌어 보여서 왔어요` | 목적 수용 (추가 캐묻기 없음) → 관심사 질문 + skip 안내 |
| 3 | `넘어갈게요` | skip 수용 → 완료 메시지 |

### 기대 결과

- interests 는 skipped/미입력 상태로 기록
- 목적이 짧아도 추가 압박 질문 없음
- `context/interests.md` 링크는 유지하되 값은 비어 있음

### Pass 기준

- [ ] `넘어갈게요` 입력 시 추가 관심사 질문 없이 진행
- [ ] profile.md의 `frontmatter.links`에 `[[context/interests]]` 포함
- [ ] 목적 `"그냥 재밌어 보여서 왔어요"` 원문 보존 (의역/재구성 없음)
- [ ] 완료 메시지 정상 출력

### Fail 기준

- [ ] skip 후 "다른 표현으로라도 말씀해주세요" 같은 재질문 존재
- [ ] 목적이 짧다고 추가 질문함
- [ ] interests 링크가 출력에서 제거됨

### 캡처할 증거

- 관심사 skip 처리된 대화 로그
- 최종 산출 데이터에서 interests 필드 상태

## QA-O-03: 재방문 사용자 (이미 profile.md 존재 시 동작)

### 목적

Memory에 이미 사용자 정보가 있을 때 중복 질문 없이 빠르게 전환하는지 확인한다.

### 전제 조건

- Memory에 아래 데이터 존재:
  ```
  profile.md:
    name: 현우
    purpose: 업무 자동화
    tech_level: medium
  context/interests.md:
    interests: ["AI", "자동화"]
  ```

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | Memory 읽기 → 이미 충분한 프로필 감지 → 즉시 전환 안내 또는 확인만 |

### 기대 결과

- 이름, 목적, 관심사를 다시 묻지 않음
- "현우님, 반가워요" 같이 기존 이름 사용
- 바로 다음 단계로 전환하거나, 변경 사항 있는지만 짧게 확인

### Pass 기준

- [ ] 이름을 다시 묻지 않음
- [ ] 목적을 다시 묻지 않음
- [ ] 관심사를 다시 묻지 않음
- [ ] 기존 닉네임(`현우`)을 사용
- [ ] 불필요한 온보딩 반복 없이 전환 또는 확인

### Fail 기준

- [ ] Memory에 있는 정보를 무시하고 처음부터 질문 시작
- [ ] 이미 아는 사실을 재질문

### 캡처할 증거

- Memory 읽기 후 첫 응답
- 전환 안내 메시지

## QA-O-04: 빈 닉네임 입력

### 목적

사용자가 빈 응답/공백/무의미 입력을 했을 때 한 번만 짧게 복구하는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `...` | 짧은 복구 메시지 (1~2문장) |
| 2 | `수아요` | 이름 수용 → 목적 질문으로 진행 |
| 3 | `정보 수집 목적으로요` | 목적 수용 → 관심사 질문 |
| 4 | `패스` | skip 수용 → 완료 |

### 기대 결과

- 복구 메시지가 짧고 부드러움 ("편하게 부를 이름 하나만 알려주세요" 수준)
- 복구는 1회만 시도
- 전체 질문 예산(3-4개)을 넘기지 않음

### Pass 기준

- [ ] 빈 입력에 대해 짧은 복구 메시지 1회만 출력
- [ ] 복구 후 정상 흐름으로 복귀
- [ ] 총 질문 수가 4개 이하
- [ ] `content.identity.name` = `"수아"`

### Fail 기준

- [ ] 빈 입력에 대해 장문 설명 또는 2회 이상 재질문
- [ ] 복구 메시지가 3문장 이상
- [ ] 전체 질문 예산 초과 (5문항 이상)

### 캡처할 증거

- 빈 입력 시 복구 메시지 전문
- 전체 턴 수

## QA-O-05: 특수문자 닉네임

### 목적

이모지, 숫자, 영문, 특수문자가 포함된 닉네임을 정상 수용하는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계 (3가지 변형)

**변형 A: 이모지 닉네임**

| 턴 | 사용자 입력 | 기대 |
|---|---|---|
| 1 | `🐱` | 이모지만 있는 경우: 복구 질문 또는 수용 (에이전트 판단에 따라) |

**변형 B: 영문+숫자 닉네임**

| 턴 | 사용자 입력 | 기대 |
|---|---|---|
| 1 | `dev42` | 정상 수용, 이후 목적 질문 |

**변형 C: 긴 닉네임**

| 턴 | 사용자 입력 | 기대 |
|---|---|---|
| 1 | `하늘을나는고양이` | 정상 수용, 이후 목적 질문 |

### 기대 결과

- 이모지 단독은 빈 응답과 유사하게 복구 가능
- 영문/숫자/긴 이름은 정상 수용
- `content.identity.name`에 사용자 입력값 그대로 저장

### Pass 기준

- [ ] 영문/숫자 닉네임을 정상 수용
- [ ] 긴 닉네임을 자르거나 변형하지 않음
- [ ] 이모지 단독 시 복구 또는 합리적 수용 (에이전트 스펙 "빈 응답, 공백, 이모지 하나만" → 복구 질문)
- [ ] 수용된 닉네임이 `content.identity.name`에 보존됨

### Fail 기준

- [ ] 유효한 닉네임을 거부함
- [ ] 닉네임을 임의로 변형/축약함
- [ ] 이모지 단독에 대한 처리 없이 그대로 진행 후 빈 이름 저장

### 캡처할 증거

- 각 변형별 수용/복구 응답
- 최종 name 필드 값

## QA-O-06: 목적 불명확 응답

### 목적

목적이 모호해도 캐묻지 않고 그대로 수용하는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `지우` | 이름 수용 → 목적 질문 |
| 2 | `그냥` | 목적 수용 (추가 질문 없음) → 관심사 질문 |
| 3 | `넘어가기` | skip → 완료 |

### 기대 결과

- 목적 `"그냥"` 을 그대로 수용
- "좀 더 구체적으로 말씀해주시겠어요?" 같은 추궁 없음
- 내부 purpose 분류: curiosity / fun 등으로 느슨하게 분류 가능

### Pass 기준

- [ ] 모호한 목적에 추가 질문 없이 다음 단계로 진행
- [ ] `content.context.purpose`에 사용자 원문 보존
- [ ] 폼처럼 캐묻는 행동 없음

### Fail 기준

- [ ] 목적이 모호하다고 세부 항목을 추궁
- [ ] 사용자 표현을 에이전트가 임의로 바꿔서 저장

### 캡처할 증거

- "그냥" 입력 시 에이전트 응답
- 최종 purpose 필드 값

## QA-O-07: 한 번에 여러 정보 제공

### 목적

사용자가 한 번에 이름과 목적을 함께 말했을 때, 중복 질문 없이 처리하는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `하은이요. 뉴스 편하게 받아보려고 왔어요.` | 이름 + 목적 동시 추출 → 관심사 질문으로 건너뛰기 |
| 2 | `기술, 스타트업` | 관심사 수용 → 완료 |

### 기대 결과

- 이미 목적을 말했으므로 "여기 오신 이유가 있으세요?" 재질문 없음
- 총 턴 수 단축 (2턴으로 완료 가능)

### Pass 기준

- [ ] 이름과 목적을 한 턴에서 모두 추출
- [ ] 이미 답한 목적을 재질문하지 않음
- [ ] `content.identity.name` = `"하은"`
- [ ] `content.context.purpose`에 목적 보존

### Fail 기준

- [ ] 이미 말한 목적을 다시 질문
- [ ] 여러 정보 중 일부를 무시

### 캡처할 증거

- 한 턴 입력 후 에이전트 응답 (어떤 질문으로 넘어갔는지)
- 최종 name + purpose 필드

## QA-O-08: 온보딩 중 범위 밖 질문

### 목적

온보딩 도중 잡담/상담/검색 요청이 들어왔을 때 적절히 거절하고 온보딩을 이어가는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `소연` | 이름 수용 → 목적 질문 |
| 2 | `오늘 날씨 어때?` | 범위 밖 인식 → 잡담 거절 + 온보딩 질문 복귀 |
| 3 | `자동화 목적이요` | 목적 수용 → 관심사 질문 |
| 4 | `넘어가기` | 완료 |

### 기대 결과

- 잡담/검색 요청에 직접 답하지 않음
- 장문 설명 없이 짧게 거절
- 온보딩 흐름이 끊기지 않고 다음 질문으로 복귀

### Pass 기준

- [ ] 날씨 질문에 직접 답하지 않음
- [ ] 거절 후 목적 질문으로 자연스럽게 복귀
- [ ] 거절 메시지가 3문장 이하
- [ ] 감정 상담 모드로 전환되지 않음

### Fail 기준

- [ ] 날씨를 직접 알려줌
- [ ] 거절 후 온보딩 흐름이 리셋됨
- [ ] 장문 사과/설명

### 캡처할 증거

- 범위 밖 질문 시 거절 응답 전문
- 거절 후 복귀한 질문 내용

## QA-O-09: 온보딩 중 에이전트 생성 요청

### 목적

온보딩 도중 사용자가 에이전트 생성을 요청해도 즉시 빌더로 전환하지 않고 온보딩을 먼저 완료하는지 확인한다.

### 전제 조건

- Memory: 비어 있음

### 입력 / 단계

| 턴 | 사용자 입력 | 기대하는 에이전트 행동 |
|---|---|---|
| 0 | (세션 시작) | 인사 + 이름 질문 |
| 1 | `테오. 그리고 경쟁사 뉴스 봇 하나 만들어줘.` | 이름 수용 + 봇 요청 인지하되 실행 거부 → 목적 질문으로 계속 |
| 2 | `경쟁사 소식 놓치기 싫어서요` | 목적 수용 → 관심사 질문 |
| 3 | `SaaS요` | 관심사 수용 → 완료 메시지 (봇 만들기를 다음 단계 힌트로 연결) |

### 기대 결과

- 봇 생성 요청을 무시하지 않지만, 온보딩 완료 전 실행하지 않음
- 다음 단계 안내에서 "경쟁사 뉴스 봇" 힌트를 포함할 수 있음
- Persona Builder 행동(사양 확정, 스케줄 설계 등)이 발생하지 않음

### Pass 기준

- [ ] 봇 생성 요청을 즉시 실행하지 않음
- [ ] "먼저 정리하고 다음 단계에서" 같은 안내 존재
- [ ] 온보딩 질문 흐름이 중단되지 않음
- [ ] 완료 메시지에 다음 단계 안내 포함
- [ ] Persona Builder 역할 침범 없음 (사양 확정, 파일 생성 약속 없음)

### Fail 기준

- [ ] 온보딩 도중 에이전트 사양 확정 시작
- [ ] "어떤 블로그를 보면 될까요?" 같은 Persona Builder 질문 등장
- [ ] 봇 생성 요청을 완전히 무시

### 캡처할 증거

- 봇 생성 요청 처리 응답 전문
- 완료 메시지의 다음 단계 안내 내용
