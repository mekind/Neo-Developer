비개발자가 겪는 4가지 벽

복잡한 설정

시나리오·조건 분기·연동 설정이
많아 30분 만에 포기

기술 용어

API, Webhook, JSON, 토큰…
"그냥 답변만 하면 되는데"

PC 상시 실행

로컬 스크립트·봇은 PC를
계속 켜둬야 해서 불편

운영·배포 부담

설치, 업데이트, 장애 대응,
호스팅 관리까지 직접 해야

타겟 페르소나

반복 리서치가 많은 실무자·마케터
Notion·Slack 능숙 / 코딩 불가
경쟁사 블로그·뉴스 수동 확인 — 매일 15분 낭비
"매번 같은 걸 물어봐야 해서 귀찮아"
반복 문의가 많은 1인 사업자·쇼핑몰 운영자
스마트스토어 익숙 / 나머지 어려움
배송·교환·반품 문의 수동 응대 — 매일 20분 낭비
"챗봇 설정이 너무 복잡해서 결국 안 씀"
AI를 써보고 싶은 비개발자·초심자
호기심은 있지만 뭘 만들지 모름
빈 화면 + 예시 없음 → 5분 만에 이탈
"AI가 뭐든 해준다던데… 근데 뭘 시켜야 하지?"

해결 접근

비개발자의 벽을 허무는 3가지 전략

🎮

게임 같은 경험

Gather 스타일 픽셀 공간
도트 캐릭터로 에이전트와 만남
설정 화면이 아닌 놀이 공간

💬

대화로 정의

"이런 거 해줘" 한 마디면 끝
Persona Builder가 의도를 파악하고
자동으로 에이전트를 생성

🧠

기억하는 시스템

llm-wiki 패턴의 Memory
한 번 말한 건 다시 안 물어봄
쓸수록 질문이 줄어드는 경험

기술 아키텍처

Frontend — Gather 스타일 UI
캐릭터 이동 · 대화 UI · 알림 렌더
▼ HTTPS
NestJS Backend — 데이터 & 제약
User · Memory · Agent CRUD | Postgres (Prisma) | Auth | 3개 제한 | 알림 저장
▼ /api/invoke | ▲ PUT memory, notifications
OpenClaw Runtime — Stateless HTTP
SOUL → 시스템 프롬프트 | 가드레일 | Skill 실행 | LLM 호출
load-agent build-prompt guardrails select-skills run write-back
▼
Vercel AI Gateway
AI SDK v6 · LLM 프로바이더 추상화
External APIs
insane-search · 검색 · 크롤링

Memory 관리

llm-wiki 패턴 — 파일 기반 지식 저장, [[link]] 연결

📂 폴더 구조

~/.myclaw/users/{user_id}/
├── index.md  ← 문서 허브
├── log.md  ← append-only 이력
├── profile.md  ← 사용자 정보
├── preferences.md
├── agents/
│   ├── index.md
│   └── {agent_id}/
│       ├── persona.md
│       ├── SOUL.md
│       └── config.md
└── context/
     ├── interests.md
     └── history.md

[[link]] 연결

문서 간 [[profile]], [[context/interests]] 형태로 연결
Persona Builder가 관련 문서를 자동 참조

append-only log

모든 변경사항이 log.md에 시간순 기록
히스토리 추적 · 디버깅 · 롤백

Memory 참조 흐름

요청 수신 → [[profile]] 읽기 → [[interests]] 읽기
→ 이미 아는 정보는 스킵 → 모르는 것만 질문
